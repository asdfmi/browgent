## Module Overview
- Sphinx-ready reStructuredText documentation set covering operators, developers, and deployers.

## Responsibilities
- Describe system architecture, execution flow, and data model.
- Provide operator guides for building, running, and monitoring workflows.
- Capture API and event references for integrators.

## Interfaces
- Index entry: `index.rst` linking to operator and developer sections.
- Key documents: `system_overview.rst`, `execution_flow.rst`, `workflow_builder_guide.rst`, `api_reference.rst`.
- Build command (if needed): `sphinx-build -b html docs build/docs` (not bundled by default).

## Depends On
- reStructuredText content files within this directory.
- External hosting (e.g., GitHub Pages) if published.
- Optional Sphinx configuration from the consumer project.

## Used By
- Developers onboarding to the architecture and APIs.
- Operators seeking workflow builder or monitoring guidance.
- Technical writers expanding the Browgent documentation set.
- GitHub Actions workflow `deploy-docs` publishing the documentation site.
